Vector Store Initialization (Qdrant)

Create a collection with named vectors and payload indexing for fast filtered retrieval.

app/core/vector_store.py 
from qdrant_client import AsyncQdrantClient
from qdrant_client.models import (
    VectorParams, Distance, HnswConfigDiff, PayloadSchemaType
)

client = AsyncQdrantClient(host="qdrant", port=6333)
COLLECTION = "knowledge_base"
VECTOR_DIM = 3072  # text-embedding-3-large

async def init_qdrant():
    collections = await client.get_collections()
    names = [c.name for c in collections.collections]

    if COLLECTION not in names:
        await client.create_collection(
            collection_name=COLLECTION,
            vectors_config=VectorParams(
                size=VECTOR_DIM,
                distance=Distance.COSINE,
                hnsw_config=HnswConfigDiff(m=16, ef_construct=200)
            )
        )
        # Index payload fields for fast filtering
        for field in ["source_type", "date", "user_id"]:
            await client.create_payload_index(
                COLLECTION, field, PayloadSchemaType.KEYWORD
            )

Document Chunking Strategy

Chunking is critical — too small loses context, too large dilutes relevance. Use recursive splitting with overlap.

app/ingestion/chunker.py 
from langchain.text_splitter import RecursiveCharacterTextSplitter
from dataclasses import dataclass
from typing import List

@dataclass
class Chunk:
    text: str
    metadata: dict
    chunk_id: str

class AdaptiveChunker:
    """Different chunk sizes per content type."""
    STRATEGIES = {
        "email":    {"chunk_size": 512,  "overlap": 50},
        "document": {"chunk_size": 1024, "overlap": 128},
        "note":     {"chunk_size": 256,  "overlap": 32},
    }

    def chunk(self, text: str, source_type: str, metadata: dict) -> List[Chunk]:
        strategy = self.STRATEGIES.get(source_type, self.STRATEGIES["document"])
        splitter = RecursiveCharacterTextSplitter(
            chunk_size=strategy["chunk_size"],
            chunk_overlap=strategy["overlap"],
            separators=["\n\n", "\n", ". ", " "]
        )
        texts = splitter.split_text(text)
        return [
            Chunk(
                text=t,
                metadata={**metadata, "chunk_index": i, "total_chunks": len(texts)},
                chunk_id=f"{metadata['doc_id']}_chunk_{i}"
            )
            for i, t in enumerate(texts)
        ]

Embedding & Indexing Pipeline

Embed chunks in parallel batches and upsert to Qdrant with full metadata payload.

app/ingestion/indexer.py 
from openai import AsyncOpenAI
from qdrant_client.models import PointStruct, UpdateStatus
from app.core.vector_store import client, COLLECTION
from app.ingestion.chunker import Chunk

oai = AsyncOpenAI()  # used only for embeddings

async def embed_batch(texts: list[str]) -> list[list[float]]:
    response = await oai.embeddings.create(
        model="text-embedding-3-large",
        input=texts,
        dimensions=3072
    )
    return [e.embedding for e in response.data]

async def index_chunks(chunks: list[Chunk], batch_size: int = 32):
    """Batch embed and upsert to Qdrant."""
    for i in range(0, len(chunks), batch_size):
        batch = chunks[i:i+batch_size]
        vectors = await embed_batch([c.text for c in batch])
        points = [
            PointStruct(
                id=c.chunk_id,
                vector=v,
                payload={"text": c.text, **c.metadata}
            )
            for c, v in zip(batch, vectors)
        ]
        result = await client.upsert(COLLECTION, points=points)
        assert result.status == UpdateStatus.COMPLETED

Hybrid Retrieval + Reranking (Advanced)

Dense vector search + Cohere reranking. This is what separates a good RAG from a great one.

app/retrieval/retriever.py 
import cohere
from qdrant_client.models import Filter, FieldCondition, MatchValue
from app.core.vector_store import client, COLLECTION
from app.ingestion.indexer import embed_batch

co = cohere.AsyncClient()

async def retrieve(
    query: str,
    top_k: int = 20,
    source_filter: str | None = None,
    rerank: bool = True
) -> list[dict]:
    [q_vec] = await embed_batch([query])

    qdrant_filter = None
    if source_filter:
        qdrant_filter = Filter(must=[
            FieldCondition("source_type", match=MatchValue(value=source_filter))
        ])

    results = await client.search(
        collection_name=COLLECTION,
        query_vector=q_vec,
        query_filter=qdrant_filter,
        limit=top_k,
        with_payload=True
    )
    docs = [r.payload for r in results]

    if rerank and len(docs) > 1:
        reranked = await co.rerank(
            query=query,
            documents=[d["text"] for d in docs],
            model="rerank-v3.5",
            top_n=5
        )
        docs = [docs[r.index] for r in reranked.results]

    return docs[:5]

Streaming Chat Endpoint — Gemini 2.5

The LLM call with context injection, source citations, and SSE streaming. Gemini 2.5 Flash is the primary provider; OpenAI is the fallback. Both are controlled by LLM_PROVIDER in your .env.

app/api/chat.py 
from fastapi import APIRouter
from fastapi.responses import StreamingResponse
from pydantic import BaseModel
import google.generativeai as genai
from openai import AsyncOpenAI
from app.retrieval.retriever import retrieve
from app.core.config import settings
import json

router = APIRouter()

# Configure Gemini with your API key from .env
genai.configure(api_key=settings.GEMINI_API_KEY)
gemini = genai.GenerativeModel("gemini-2.5-flash")
oai = AsyncOpenAI()  # fallback

SYSTEM_PROMPT = """You are a personal knowledge assistant.
Answer questions using ONLY the provided context.
Always cite sources using [Source N] format.
If context is insufficient, say so clearly."""

class ChatRequest(BaseModel):
    message: str
    source_filter: str | None = None  # e.g. "email", "document", "note"

@router.post("/stream")
async def stream_chat(body: ChatRequest):
    async def generate():
        docs = await retrieve(body.message, top_k=20, source_filter=body.source_filter)

        # Emit source metadata first so the UI can show cards immediately
        yield (f"data: {json.dumps({'type':'sources','data':docs})}\n\n").encode()

        context = "\n\n".join([
            f"[Source {i+1}] {d['source_type'].upper()} — {d['title']}\n{d['text']}"
            for i, d in enumerate(docs)
        ])
        prompt = f"{SYSTEM_PROMPT}\n\nContext:\n{context}\n\nQuestion: {body.message}"

        if settings.LLM_PROVIDER == "gemini":
            # Gemini 2.5 native streaming — uses your GEMINI_API_KEY from .env
            response = await gemini.generate_content_async(
                prompt,
                generation_config={"temperature": 0.2, "max_output_tokens": 2048},
                stream=True
            )
            async for chunk in response:
                delta = chunk.text or ""
                if delta:
                    yield (f"data: {json.dumps({'type':'token','data':delta})}\n\n").encode()
        else:
            # OpenAI fallback
            stream = await oai.chat.completions.create(
                model="gpt-4o", stream=True,
                messages=[{"role": "user", "content": prompt}]
            )
            async for chunk in stream:
                delta = chunk.choices[0].delta.content or ""
                if delta:
                    yield (f"data: {json.dumps({'type':'token','data':delta})}\n\n").encode()

        yield b"data: [DONE]\n\n"

    return StreamingResponse(generate(), media_type="text/event-stream")

Gmail Ingestion via OAuth 2.0

Use Google's official Python client. OAuth flow gives a token; fetch emails and process them. Includes the _extract_body helper for multipart MIME parsing.

app/ingestion/sources/gmail.py 
from google.oauth2.credentials import Credentials
from googleapiclient.discovery import build
from app.ingestion.chunker import AdaptiveChunker
from app.ingestion.indexer import index_chunks
import base64, email as email_lib

def _extract_body(payload: dict) -> str:
    """Recursively extract plain-text body from a Gmail MIME payload."""
    mime_type = payload.get("mimeType", "")
    body_data = payload.get("body", {}).get("data", "")

    if mime_type == "text/plain" and body_data:
        return base64.urlsafe_b64decode(body_data + "==").decode("utf-8", errors="replace")

    for part in payload.get("parts", []):
        text = _extract_body(part)
        if text:
            return text

    return ""

async def ingest_gmail(credentials: Credentials, max_emails: int = 500) -> dict:
    service = build("gmail", "v1", credentials=credentials)
    chunker = AdaptiveChunker()

    messages = service.users().messages().list(
        userId="me", maxResults=max_emails, q="in:inbox"
    ).execute().get("messages", [])

    all_chunks = []
    for msg_ref in messages:
        msg = service.users().messages().get(
            userId="me", id=msg_ref["id"], format="full"
        ).execute()

        headers = {h["name"]: h["value"] for h in msg["payload"]["headers"]}
        body = _extract_body(msg["payload"])
        if not body.strip():
            continue

        metadata = {
            "doc_id": msg_ref["id"],
            "source_type": "email",
            "title": headers.get("Subject", "(no subject)"),
            "from": headers.get("From", ""),
            "date": headers.get("Date", ""),
        }
        all_chunks.extend(chunker.chunk(body, "email", metadata))

    await index_chunks(all_chunks)
    return {"indexed": len(all_chunks), "emails": len(messages)}

PDF & Markdown File Ingestion

Use LangChain's document loaders for diverse file types.

app/ingestion/sources/files.py 
from langchain_community.document_loaders import (
    PyPDFLoader, UnstructuredMarkdownLoader,
    TextLoader, Docx2txtLoader
)
from app.ingestion.chunker import AdaptiveChunker
from app.ingestion.indexer import index_chunks
from pathlib import Path

LOADERS = {
    ".pdf":  PyPDFLoader,
    ".md":   UnstructuredMarkdownLoader,
    ".txt":  TextLoader,
    ".docx": Docx2txtLoader,
}

async def ingest_file(file_path: str) -> int:
    path = Path(file_path)
    loader_cls = LOADERS.get(path.suffix.lower())
    if not loader_cls:
        raise ValueError(f"Unsupported file type: {path.suffix}")

    pages = loader_cls(file_path).load()
    chunker = AdaptiveChunker()
    all_chunks = []

    for page in pages:
        metadata = {
            "doc_id": f"file_{path.stem}",
            "source_type": "document",
            "title": path.name,
            "page": page.metadata.get("page", 0),
        }
        all_chunks.extend(chunker.chunk(page.page_content, "document", metadata))

    await index_chunks(all_chunks)
    return len(all_chunks)

Notion Ingestion

Pull pages from your Notion workspace via the official API. Set your NOTION_API_KEY in .env — create an integration at notion.com/my-integrations and share your pages with it.

app/ingestion/sources/notion.py 
from notion_client import AsyncClient
from app.ingestion.chunker import AdaptiveChunker
from app.ingestion.indexer import index_chunks
from app.core.config import settings

notion = AsyncClient(auth=settings.NOTION_API_KEY)

def _blocks_to_text(blocks: list) -> str:
    """Flatten Notion block objects to plain text."""
    lines = []
    for block in blocks:
        btype = block.get("type", "")
        rich = block.get(btype, {}).get("rich_text", [])
        text = "".join([r.get("plain_text", "") for r in rich])
        if text:
            lines.append(text)
    return "\n".join(lines)

async def ingest_notion_page(page_id: str) -> int:
    # Fetch page metadata
    page = await notion.pages.retrieve(page_id=page_id)
    title_prop = page["properties"].get("title", page["properties"].get("Name", {}))
    title = title_prop.get("title", [{}])[0].get("plain_text", "Untitled")

    # Fetch all blocks (handles pagination)
    all_blocks = []
    cursor = None
    while True:
        resp = await notion.blocks.children.list(
            block_id=page_id, start_cursor=cursor
        )
        all_blocks.extend(resp["results"])
        if not resp["has_more"]:
            break
        cursor = resp["next_cursor"]

    text = _blocks_to_text(all_blocks)
    if not text.strip():
        return 0

    metadata = {
        "doc_id": f"notion_{page_id}",
        "source_type": "note",
        "title": title,
        "page_id": page_id,
        "url": page.get("url", ""),
    }
    chunker = AdaptiveChunker()
    chunks = chunker.chunk(text, "note", metadata)
    await index_chunks(chunks)
    return len(chunks)

async def ingest_notion_search(query: str = "") -> dict:
    """Ingest all pages your integration has access to."""
    results = await notion.search(query=query, filter={"value": "page", "property": "object"})
    total_chunks = 0
    for page in results["results"]:
        total_chunks += await ingest_notion_page(page["id"])
    return {"pages": len(results["results"]), "chunks": total_chunks}

useChat Hook — SSE Streaming

Custom React hook that handles Server-Sent Events, assembles tokens, and tracks sources. Includes the Source interface.

src/hooks/useChat.ts 
import { useState, useCallback, useRef } from 'react'

// Matches the payload stored in Qdrant for each chunk
interface Source {
  source_type: 'email' | 'document' | 'note'
  title: string
  date?: string
  from?: string
  page?: number
  url?: string
}

interface Message {
  role: 'user' | 'assistant'
  content: string
  sources?: Source[]
  isStreaming?: boolean
}

export function useChat() {
  const [messages, setMessages] = useState<Message[]>([])
  const [isLoading, setIsLoading] = useState(false)
  const abortRef = useRef<AbortController | undefined>()

  const sendMessage = useCallback(async (text: string) => {
    setMessages(m => [...m, { role: 'user', content: text }])
    setIsLoading(true)

    abortRef.current = new AbortController()
    const res = await fetch('/api/chat/stream', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ message: text }),
      signal: abortRef.current.signal,
    })

    const reader = res.body!.getReader()
    const decoder = new TextDecoder()
    let assistantMsg: Message = { role: 'assistant', content: '', isStreaming: true }
    setMessages(m => [...m, assistantMsg])

    while (true) {
      const { done, value } = await reader.read()
      if (done) break

      const lines = decoder.decode(value).split('\n').filter(Boolean)
      for (const line of lines) {
        if (!line.startsWith('data: ')) continue
        const raw = line.slice(6)
        if (raw === '[DONE]') { setIsLoading(false); break }

        const event = JSON.parse(raw)
        if (event.type === 'sources') {
          assistantMsg = { ...assistantMsg, sources: event.data as Source[] }
        } else if (event.type === 'token') {
          assistantMsg = { ...assistantMsg, content: assistantMsg.content + event.data }
        }
        setMessages(m => [...m.slice(0, -1), { ...assistantMsg }])
      }
    }
  }, [])

  const abort = useCallback(() => abortRef.current?.abort(), [])

  return { messages, isLoading, sendMessage, abort }
}

Chat Component with Source Cards

src/components/ChatWindow.tsx 
import ReactMarkdown from 'react-markdown'
import { useChat } from '../hooks/useChat'
import { useState, useRef, useEffect } from 'react'

export function ChatWindow() {
  const { messages, isLoading, sendMessage, abort } = useChat()
  const [input, setInput] = useState('')
  const bottomRef = useRef<HTMLDivElement>(null)

  useEffect(() => {
    bottomRef.current?.scrollIntoView({ behavior: 'smooth' })
  }, [messages])

  const handleSend = () => {
    if (!input.trim() || isLoading) return
    sendMessage(input)
    setInput('')
  }

  return (
    <div className="chat-container">
      <div className="messages">
        {messages.map((msg, i) => (
          <div key={i} className={`message ${msg.role}`}>
            <ReactMarkdown>{msg.content}</ReactMarkdown>
            {msg.isStreaming && <span className="cursor" />}
            {msg.sources && msg.sources.length > 0 && (
              <div className="sources">
                <span className="sources-label">Sources</span>
                {msg.sources.map((src, j) => (
                  <div key={j} className="source-card">
                    <span className="source-type">{src.source_type}</span>
                    <span className="source-title">{src.title}</span>
                    {src.date && <span className="source-date">{src.date}</span>}
                  </div>
                ))}
              </div>
            )}
          </div>
        ))}
        <div ref={bottomRef} />
      </div>
      <div className="input-bar">
        <input
          value={input}
          onChange={e => setInput(e.target.value)}
          onKeyDown={e => e.key === 'Enter' && handleSend()}
          placeholder="Ask anything about your knowledge base..."
          disabled={isLoading}
        />
        {isLoading
          ? <button onClick={abort} className="btn-stop">Stop</button>
          : <button onClick={handleSend}>Send</button>
        }
      </div>
    </div>
  )
}

Encrypting Personal Data at Rest

All document text stored in Qdrant payloads is encrypted with AES-256-GCM before indexing.

app/core/encryption.py 
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os, base64

class DataEncryptor:
    def __init__(self, key: bytes | None = None):
        # Key from env var — never hardcoded. Generate with:
        # python -c "import secrets; print(secrets.token_hex(32))"
        self.key = key or bytes.fromhex(os.environ["ENCRYPTION_KEY"])
        self.aesgcm = AESGCM(self.key)

    def encrypt(self, plaintext: str) -> str:
        nonce = os.urandom(12)  # 96-bit nonce for GCM
        ct = self.aesgcm.encrypt(nonce, plaintext.encode(), None)
        return base64.b64encode(nonce + ct).decode()

    def decrypt(self, ciphertext: str) -> str:
        data = base64.b64decode(ciphertext)
        nonce, ct = data[:12], data[12:]
        return self.aesgcm.decrypt(nonce, ct, None).decode()

JWT Auth + Rate Limiting

Protect API endpoints and prevent abuse of LLM calls (which cost money).

app/core/auth.py 
from fastapi import Depends, HTTPException
from fastapi.security import HTTPBearer
from jose import jwt, JWTError
from datetime import datetime  # FIX: explicit import
import redis.asyncio as aioredis
from app.core.config import settings

security = HTTPBearer()
redis_client = aioredis.from_url("redis://redis:6379")

async def get_current_user(token = Depends(security)) -> dict:
    try:
        payload = jwt.decode(
            token.credentials, settings.JWT_SECRET, algorithms=["HS256"]
        )
        return payload
    except JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")

async def rate_limit(user = Depends(get_current_user)):
    # Sliding window: 20 requests per minute per user
    key = f"rate:{user['sub']}:{datetime.now().strftime('%H%M')}"
    count = await redis_client.incr(key)
    if count == 1:
        await redis_client.expire(key, 60)
    if count > 20:
        raise HTTPException(status_code=429, detail="Rate limit exceeded")

OAuth Token Storage (Never in plaintext)

Store Gmail OAuth refresh tokens encrypted in PostgreSQL. The build_credentials helper constructs a valid Google credentials object from the decrypted token.

app/core/token_store.py 
from sqlalchemy.ext.asyncio import AsyncSession
from google.oauth2.credentials import Credentials
from app.core.encryption import DataEncryptor
from app.models.user import OAuthToken
from app.core.config import settings

enc = DataEncryptor()

def build_credentials(refresh_token: str) -> Credentials:
    """Reconstruct a Google OAuth Credentials object from a stored refresh token."""
    return Credentials(
        token=None,
        refresh_token=refresh_token,
        token_uri="https://oauth2.googleapis.com/token",
        client_id=settings.GOOGLE_CLIENT_ID,
        client_secret=settings.GOOGLE_CLIENT_SECRET,
    )

async def store_google_token(user_id: str, refresh_token: str, db: AsyncSession):
    # Encrypt before storing — never write plaintext tokens to DB
    token_record = OAuthToken(
        user_id=user_id,
        provider="google",
        encrypted_token=enc.encrypt(refresh_token)
    )
    db.add(token_record)
    await db.commit()

async def get_google_credentials(user_id: str, db: AsyncSession) -> Credentials:
    record = await db.get(OAuthToken, user_id)
    if not record:
        raise ValueError(f"No OAuth token for user {user_id}")
    refresh_token = enc.decrypt(record.encrypted_token)
    return build_credentials(refresh_token)

Presenting This to Google

How to frame this project to maximize interview impact

🚀
The Narrative: "I built a production-grade RAG system that ingests personal data from Gmail, Notion, and PDFs. I chose Gemini 2.5 Flash as the primary LLM for its 1M token context window — which lets me pass all retrieved chunks without truncation, a real architectural advantage. I evaluated retrieval quality with RAGAS and improved precision from 60% to 87% by adding Cohere reranking. The whole stack deploys to GCP Cloud Run with CI/CD via GitHub Actions."

What interviewers will ask — and what to say

  • Q
    "Why Gemini 2.5 over GPT-4o?"
    Answer: Gemini 2.5 Flash has a 1M token context window vs GPT-4o's 128K. For RAG, this means I can pass all retrieved chunks without truncation. It's also faster, cheaper per token, and natively integrates with GCP infrastructure. I kept OpenAI as a drop-in fallback via an LLM_PROVIDER env flag — demonstrating that the architecture isn't vendor-locked.
  • Q
    "How does your search scale to millions of documents?"
    Answer: HNSW index in Qdrant is O(log n) for search. For >10M vectors, switch to Qdrant's distributed mode or add scalar quantization to reduce memory footprint by 4x. At true Google scale, I'd migrate to Vertex AI Matching Engine.
  • Q
    "How do you handle stale data / document updates?"
    Answer: Each document has a content hash. On re-ingest, compare hash — if changed, delete old chunks by doc_id filter and re-index. Qdrant supports filtering deletes. Gmail/Notion webhooks can trigger this automatically.
  • Q
    "How did you evaluate whether RAG answers are good?"
    Answer: Used RAGAS framework — measures faithfulness (answer grounded in context), answer relevancy, context precision/recall. Created 50 test Q&A pairs from my own data. Reranking improved context precision from 60% to 87%.
  • Q
    "What would you do differently at Google scale?"
    Answer: Move to Spanner for metadata, Vertex AI Matching Engine for vectors, Pub/Sub for ingestion queue, Cloud Workflows for orchestration, Gemini on Vertex AI for enterprise SLAs, and proper SLOs with Cloud Monitoring.
  • Q
    "How do you prevent prompt injection attacks?"
    Answer: System prompt clearly separates context from user input with structural delimiters. Input sanitization strips injection patterns. Gemini 2.5 is instructed to only answer from provided context, and I validate that citations in the response map to actual retrieved sources.
